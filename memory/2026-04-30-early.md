Or when a template maintainer removes a task version from the library (e.g., `Docker@1` deprecated in favor of `Docker@2`), pipelines referencing the old version continue to queue but the task no longer exists, causing silent failures that don't surface until the pipeline hits that step. Azure DevOps shows pipeline run status but doesn't track which pipelines consume which template versions, doesn't detect when template changes have been applied across the organization, or forecast when a template drift will cause silent task version regressions across hundreds of pipelines simultaneously. Teams discover the problem only when a widespread pipeline failure is reported or when a security regression is discovered and the investigation reveals a template update was pushed to production pipelines without testing.

**Target Market:** Mid-large SaaS on Azure DevOps with YAML CI/CD pipelines, shared template libraries across multiple teams, multi-environment deployments (dev/staging/prod), and strict pipeline change control requirements.

**Technical Architecture:**
- Python/Go collector via Azure DevOps REST API (Pipelines, PipelineTemplates, Tasks, TaskGroups) + Azure DevOps Audit Logs (template update events, pipeline run events) + Azure DevOps YAML pipeline source (Git repos containing templates) + Azure Pipelines Agent events
- Template consumption graph builder: traverses all YAML pipeline repos to build a complete map of which pipelines include/extend which templates at which versions, maintains version pinning vs. floating references
- Template drift detector: monitors Azure DevOps audit logs for template file changes, captures the exact diff of template modifications (task version changes, parameter additions/removals, condition logic changes), flags when shared templates have drifted from their last-known-good state
- Silent task version regression forecaster: when a template updates a task reference (e.g., `Docker@1` → `Docker@2`), identifies all affected pipelines, predicts which pipelines will fail or behave differently based on task version behavior differences
- Template version chaos analyzer: for pipelines using floating template references (`template.yml@refs/heads/main` vs. pinned `template.yml@refs/tags/v2.3.1`), monitors when `main` branch template changes and which pipelines will automatically pick up the change on next run
- Pipeline impact radius calculator: for each template change, calculates the blast radius — how many pipelines will be affected, which environments (dev/staging/prod), which teams will be impacted, and which pipelines have no owners assigned
- Deprecated task version detector: monitors Azure DevOps Marketplace for task version deprecations, cross-references with organization pipeline template usage, flags when pipelines are using tasks that will stop working after deprecation date
- Azure DevOps API + PostgreSQL + Grafana: template drift timeline per template, task version regression risk heatmap per pipeline, template consumption graph visualization, deprecated task usage inventory, blast radius matrix per template change
- Alerting: Slack with "Azure DevOps YAML template drift alert: Template `shared/container-build.yml` updated 15 min ago (commit `abc123` by `platform-team`). Change: `Docker@1` → `Docker@2` (breaking: `--tag` flag syntax changed). Impact: 47 pipelines across 8 teams will be affected on next run. Pipelines at immediate risk: `api-service-prod` (refs/heads/main, runs every 4h, next in 20 min), `worker-prod-deploy` (refs/heads/main). Task version regression expected: Docker build commands will fail with unknown flag error. Pipelines using pinned version `refs/tags/v3.1.0` are safe. Recommend: test template change in `pipeline-check` org before merging to main, or pin affected pipelines to current template tag."
